# Sound Academy Master Phase Roadmap

This roadmap governs the production build sequence. Phase 0 must be completed before broad lesson generation, major UI expansion, Digital Twin work or production launch.

## PHASE 0: Curriculum Architecture

Goal: create the educational operating system for Sound Academy.

Required outputs:

- Curriculum Blueprint
- Certification Blueprint
- Competency Blueprint
- Assessment Blueprint
- Learning Science Blueprint
- Specialist Pathways
- Curriculum Database Architecture
- Platform Integration Blueprint
- Curriculum Governance

Hard rule: do not generate hundreds of lessons until the academy hierarchy, certifications, competencies, progression rules and assessment architecture are approved.

## PHASE 1: Repository Audit

Goal: confirm current app structure, technical debt, route coverage, schema gaps, dependency health and Vercel readiness.

Outputs:

- Route/component audit
- Schema audit
- Content audit
- Risk register
- Backlog with priorities

Status: initial audit completed in [phase-1/REPOSITORY_AUDIT.md](phase-1/REPOSITORY_AUDIT.md).

## PHASE 2: Design System

Goal: establish the premium Sound Academy visual language.

Outputs:

- Dark control-room tokens
- Glass panel rules
- Neon accent system
- Mobile service-mode design rules
- Interaction state library
- Legal-safe X32/Logic/Dante mock guidelines

Status: initial design system completed in [phase-2/DESIGN_SYSTEM_BLUEPRINT.md](phase-2/DESIGN_SYSTEM_BLUEPRINT.md).

## PHASE 3: Platform Architecture

Goal: align app architecture with curriculum graph, service operations, assets, evidence and production deployment.

Outputs:

- Postgres migration plan

- Asset storage plan

- CI/CD and Vercel preview plan

Status: initial platform data spine completed in [phase-3/PLATFORM_ARCHITECTURE_BLUEPRINT.md](phase-3/PLATFORM_ARCHITECTURE_BLUEPRINT.md).

## PHASE 4: Academy Experience

Goal: make Academy a skill roadmap rather than a module list.

Outputs:

- Skill tree UI

- Module pages

Status: initial academy pathway experience completed in [phase-4/ACADEMY_EXPERIENCE_BLUEPRINT.md](phase-4/ACADEMY_EXPERIENCE_BLUEPRINT.md).

## PHASE 5: Practical Training Systems

Goal: build task-based learning for X32, Logic, Dante, Waves, wireless, P16 and troubleshooting.

Status: initial practical training workflow engine completed in [phase-5/PRACTICAL_TRAINING_SYSTEMS_BLUEPRINT.md](phase-5/PRACTICAL_TRAINING_SYSTEMS_BLUEPRINT.md).

Goal: make Sound Lab a diagnosis and ear-training simulator.

Status: initial interactive diagnostic Sound Lab completed in [phase-6/SOUND_LAB_BLUEPRINT.md](phase-6/SOUND_LAB_BLUEPRINT.md).
