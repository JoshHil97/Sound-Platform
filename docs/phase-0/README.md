# Phase 0 Curriculum Architecture

Phase 0 is the educational foundation for Sound Academy. It exists to prevent the platform from becoming a polished interface with disconnected training content.

All future work should reference these documents before generating lessons, changing certifications, expanding Sound Lab, building Service Mode workflows or implementing the Digital Twin.

## Deliverables

- [Curriculum Blueprint](CURRICULUM_BLUEPRINT.md): master academy structure, module rules, lesson architecture and specialisations.
- [Certification Blueprint](CERTIFICATION_BLUEPRINT.md): certification missions, requirements, renewal rules and progression gates.
- [Competency Blueprint](COMPETENCY_BLUEPRINT.md): knowledge, practical, operational, troubleshooting, communication and leadership competency map.
- [Assessment Blueprint](ASSESSMENT_BLUEPRINT.md): knowledge, practical, scenario, service, mentor and certification assessment structure.
- [Learning Science Blueprint](LEARNING_SCIENCE_BLUEPRINT.md): active recall, spaced repetition, deliberate practice and scenario-learning model.
- [Specialist Pathways](SPECIALIST_PATHWAYS.md): X32, Dante, Logic, Waves, livestream, wireless, P16, choir and troubleshooting paths.
- [Curriculum Database Architecture](CURRICULUM_DATABASE_ARCHITECTURE.md): proposed curriculum, competency, evidence and progression models.
- [Integration Blueprint](INTEGRATION_BLUEPRINT.md): how curriculum connects to Dashboard, Academy, Sound Lab, Service Mode, Troubleshooting, Equipment, SOPs, Certifications and Digital Twin.
- [Curriculum Governance](CURRICULUM_GOVERNANCE.md): versioning, review, approval, ownership, retirement and quality standards.

## Phase 0 Agent Contributions

- Agent A, Curriculum Architect: academy hierarchy, module structure, lesson map and certification pathways.
- Agent B, Church Sound Subject Matter Expert: X32, Dante, Logic, Waves, livestream, wireless, P16, choir and troubleshooting specialist paths.
- Agent C, Learning Science Specialist: recall, spaced repetition, deliberate practice, scenario flow and mentor review design.
- Agent D, Database Architect: curriculum, competency, progression, evidence and sign-off data architecture.
- Agent E, Platform Architect: product integration, admin strategy, Digital Twin alignment and launch sequencing.

## Hard Rule

Do not generate hundreds of lessons until:

- Curriculum architecture is approved.
- Certification requirements are approved.
- Competencies are mapped.
- Progression paths are defined.
- Assessment architecture exists.

After that approval point, lesson content can be generated and populated through the Admin CMS/content pipeline.
