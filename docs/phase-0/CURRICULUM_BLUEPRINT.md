# Sound Academy Curriculum Blueprint

## Purpose

Sound Academy is not a generic LMS. It is a church production training and operations system. The curriculum is governed by service competence: volunteers must be able to explain the signal path, operate safely, diagnose faults calmly, document changes and serve real services with mentor oversight.

This blueprint governs all future lesson generation, UI design, assessment design, Sound Lab scenarios, Service Mode workflows, certifications and admin content.

## Master Academy Structure

### Foundations Academy

Mission: give new volunteers a safe mental model of ministry purpose, sound, signal flow, gain, routing and first-line troubleshooting.

Primary outcomes:

- Explain why church sound is ministry and service support.
- Trace source to microphone or DI, stagebox, X32, buses, Dante, Logic and livestream output.
- Distinguish gain, fader, bus, matrix, P16 feed and stream output.
- Perform a basic line-check and explain where signal stops.

Core modules:

- Ministry, Safety and Listening Priorities
- Foundations of Sound and Signal Flow
- Sources, Microphones, DI and Stage Inputs
- Gain, Meters and Headroom
- X32 Channel Strip Orientation
- Main LR, Buses, Matrices and P16 Basics
- Dante and Logic Path Overview
- Beginner Troubleshooting Mindset

### FOH Operator Academy

Mission: train reliable Sunday operators who can prepare, line-check, mix speech and worship, support monitors and recover common faults without panic.

Primary outcomes:

- Prepare the X32 for service from an approved showfile.
- Patch, name and gain sources.
- Mix speech and worship for FOH with safe use of EQ, gates, compression and FX.
- Build monitor and P16 sends without breaking FOH.
- Manage wireless microphones, choir microphones and talkback.
- Log service issues and hand over accurately.

Core modules:

- Sunday Service Preparation
- X32 Patch, Name and Gain
- Speech-First FOH Mixing
- Worship Team Balance
- EQ, Gate, Compression and FX on X32
- P16, Wedges and Talkback
- Wireless and Choir Operation
- Live FOH Troubleshooting

### Livestream Academy

Mission: train operators to build and run a dedicated livestream workflow that is not a copy of FOH.

Primary outcomes:

- Explain FOH versus livestream translation.
- Verify Dante/DVS into Logic.
- Use the approved Logic template.
- Build speech, vocal, choir, worship, audience and master stream paths.
- Apply Waves/Logic processing by purpose.
- Monitor loudness, latency and output handoff to OBS or encoder.

Core modules:

- FOH vs Livestream Listening
- Dante Into Logic Safely
- Logic Template Architecture
- Stream Buses and Auxes
- Speech and Vocal Clarity
- Waves Plugin Chains by Purpose
- Loudness, Limiting and Translation
- OBS/Encoder Handoff and Stream Troubleshooting

### Systems Academy

Mission: train engineers to own routing, network audio, virtual soundcheck, system documentation and fault isolation.

Primary outcomes:

- Configure X32 routing, scenes, snippets, matrices and Dante card outputs safely.
- Read Dante Controller routing, status, clocking and latency.
- Use recordings and virtual soundcheck for coaching.
- Troubleshoot cross-domain faults from source to stream.

Core modules:

- Advanced X32 Routing
- Scenes, Snippets and Rollback
- Dante Controller and DVS
- Network Audio Fault Isolation
- Recording and Virtual Soundcheck
- System Documentation and Change Logs
- Full-Service Fault Simulation

### Leadership Academy

Mission: prepare senior engineers to coach, assess, document and guard service quality.

Primary outcomes:

- Mentor volunteers with consistent rubrics.
- Approve practical sign-offs.
- Lead service debriefs.
- Maintain SOP quality.
- Manage incident review and improvement cycles.

Core modules:

- Coaching and Mentor Practice
- Assessment and Evidence Review
- SOP Writing and Knowledge Preservation
- Service Debrief Leadership
- Quality Control and Mix Review

### Technical Director Academy

Mission: govern systems, people, architecture, budget, procurement and change management.

Primary outcomes:

- Own network, console and livestream architecture.
- Govern certification and permissions.
- Manage inventory lifecycle and procurement.
- Approve routing, template and SOP changes.
- Plan redundancy and production readiness.

Core modules:

- Production System Architecture
- Network and Redundancy Governance
- Certification Governance
- Inventory, Budget and Procurement
- Change Management and Release Control
- Platform Administration and Data Stewardship

## Specialisations

- X32 Specialist: channel strip, routing, buses, matrices, scenes, snippets, P16, talkback, Dante card.
- Dante Specialist: device discovery, subscriptions, clocking, sample rate, latency, DVS, network interface, route documentation.
- Logic Specialist: templates, audio device setup, inputs, channel strips, auxes, buses, plugins, loudness, latency, OBS handoff.
- Waves Specialist: dynamic EQ, compression, multiband control, de-essing, vocal chains, bus chains, limiting and metering.
- Wireless Specialist: batteries, RF/audio checks, labelling, handoff, pastor mic discipline, failure recovery.
- Troubleshooting Specialist: symptom triage, safe first checks, signal-path diagnosis, escalation, incident logging.
- Worship Mixing Specialist: band/vocal balance, transitions, FX, dynamics, congregation support.
- Speech Specialist: intelligibility, pastor mic gain, clarity, de-essing, consistency and livestream translation.

## Module Architecture Rule

Every module must define:

- Mission and service relevance
- Learning outcomes
- Knowledge competencies
- Practical competencies
- Operational competencies
- Assessments
- Required evidence
- Mentor requirements
- Related SOPs
- Related equipment
- Related troubleshooting flows
- Related Sound Lab exercises

## Lesson Architecture Rule

Lessons are generated only after module and competency mapping is complete.

Each lesson must include:

- Objective
- Lesson type: concept, walkthrough, simulator, listening drill, service drill, scenario, mentor assessment
- Estimated duration
- Prerequisites
- Step-by-step actions
- What to press/click/check
- Why it matters
- Listening targets
- Practical exercise
- Knowledge check
- Scenario drill
- Active recall prompt
- Spaced repetition prompt
- Mentor sign-off task
- Evidence requirement
- Related SOP/equipment/troubleshooting/Sound Lab links

## Initial Lesson Map Pattern

Do not write full lesson content yet. Use this map pattern:

- Foundations module lessons are concept plus trace-and-explain drills.
- FOH module lessons are console walkthrough plus live-practice drills.
- Livestream module lessons are Logic/Dante workflow plus A/B listening drills.
- Systems module lessons are simulator plus fault-isolation drills.
- Leadership module lessons are observation, feedback and rubric drills.
- Technical Director lessons are governance case studies and approval workflows.
