# Sound Academy Assessment Blueprint

## Assessment Types

### Knowledge Assessments

Purpose: confirm the learner understands language and concepts.

Examples:

- Signal flow quiz
- Gain versus fader quiz
- Dante green tick and clocking quiz
- Logic aux/send/output quiz

Rules:

- Questions must be applied, not trivia.
- Use short quizzes after lessons.
- Use cumulative checks before certification.

### Practical Assessments

Purpose: confirm the learner can perform a task.

Examples:

- Set gain on X32 speech source
- Patch, name and line-check four sources
- Build P16 send for a worship vocalist
- Verify Logic template input/output path
- Restore Dante subscription

Rules:

- Mentor observes live or via submitted evidence.
- Must include pass/retry and notes.
- Must record service context.

### Scenario Assessments

Purpose: confirm diagnosis under pressure.

Examples:

- Pastor mic distorted
- FOH works but stream silent
- Dante device visible but no audio
- P16 user cannot hear themselves
- Choir mics feeding back

Rules:

- Learner must state symptom, likely fault domain, first safe check and escalation point.
- Sound Lab scenarios can feed scenario scores.

### Service Assessments

Purpose: confirm real-world readiness.

Examples:

- Shadow a rehearsal
- Run line check under supervision
- Operate FOH for one service segment
- Operate livestream mix under mentor review
- Complete post-service log

Rules:

- Independent operation requires multiple observed services.
- Service observations expire if the volunteer is inactive too long.

### Mentor Assessments

Purpose: standardise human judgment.

Mentor records:

- Task
- Evidence
- Risk level
- Coaching notes
- Decision
- Renewal date

### Certification Assessments

Purpose: combine knowledge, practical, scenario, service and mentor evidence.

Certification board should see:

- Module completion
- Quiz scores
- Practical sign-offs
- Sound Lab scores
- Service experience
- Mentor notes
- Expiry/renewal status

## Module Assessment Template

Every module must define:

- Required quiz
- Required practical
- Required scenario
- Required evidence
- Mentor role required
- Service requirement if any
- Retry conditions

## Initial Assessment Matrix

Foundations:

- Knowledge: signal flow and gain quiz
- Practical: trace paths and set one gain
- Scenario: silent mic first checks
- Service: one observation

FOH Operator:

- Knowledge: X32/P16/wireless quiz
- Practical: line check and FOH mix segment
- Scenario: feedback and no-mic recovery
- Service: two observed services

Livestream:

- Knowledge: Dante/Logic/loudness quiz
- Practical: Logic template and DVS setup
- Scenario: stream no sound and clipping
- Service: one rehearsal and one live stream

Systems:

- Knowledge: routing/clocking/latency quiz
- Practical: Dante restore and X32 routing documentation
- Scenario: timed cross-domain faults
- Service: supervise one service recovery

Leadership:

- Knowledge: rubric and SOP standards
- Practical: mentor another trainee
- Scenario: service debrief and escalation
- Service: lead one debrief

Technical Director:

- Knowledge: governance review
- Practical: approve change request
- Scenario: redundancy/routing change case
- Service: quarterly review
