# Phase 8 Digital Twin

Phase 8 adds the first shared model of the church production system.

## Documents

- [Digital Twin Blueprint](DIGITAL_TWIN_BLUEPRINT.md): model scope, seeded system map, route integration and future governance.

## Phase 8 Outcome

The app now has:

- Digital Twin route at `/digital-twin`
- X32 input channel model
- X32 bus model
- Dante device and subscription model
- Logic stream template channel model
- P16 source order model
- Stage zone model
- Wireless assignment model
- End-to-end digital signal paths
- Signal Flow, Equipment, X32 and Dante pages connected to twin data

This phase still uses seeded placeholders. The next maturity step is Admin CMS editing and church-owned verification against the real patch list.
